 Test Plan – SauceDemo UI Testing Project

1. Project Overview

This test plan covers the manual UI testing of the SauceDemo web application.
The goal is to validate the functionality of login, products list, cart, and checkout modules.

Website Under Test:
https://www.saucedemo.com


---

2. Scope of Testing

✔ In Scope:

Login validation (positive & negative)

Product list page

Sorting functionality

Add/Remove items from cart

Cart page

Checkout process

Input validation

UI consistency


✖ Out of Scope:

Backend validation

API testing

Performance testing

Security testing

Localization testing



---

3. Test Objectives

Ensure that users can successfully log in with valid credentials.

Validate proper error messages for invalid or missing inputs.

Verify product display, sorting, and cart behavior.

Confirm successful completion of checkout flow.

Identify functional and UI bugs that affect user experience.



---

4. Test Deliverables

TestCases_SauceDemo.xlsx

BugReport_SauceDemo.docx

TestPlan_SauceDemo.md

Screenshots (evidence)

Test summary (optional)



---

5. Test Approach

Functional Testing for UI operations

Negative Testing for invalid inputs

Exploratory Testing to find unexpected behavior

Regression Testing after major issues are fixed



---

6. Test Environment

Browser: Chrome (latest version)

OS: Windows / macOS / Android

Tools Used:

Browser DevTools

Postman (if needed)

GitHub for documentation




---

7. Entry Criteria

Test environment is ready

Website is accessible

Test cases are created

Required test data is prepared



---

8. Exit Criteria

All high-priority test cases executed

All critical bugs resolved or accepted

Test evidence uploaded

Test report completed


---

9. Risks & Mitigation

Risk	Mitigation

Website instability	Retry tests / log evidence
UI inconsistent behavior	Detailed bug reporting
Browser differences	Cross-browser validation


---

10. Testing Schedule

Test Case creation: 1 day

Execution: 1–2 days

Bug logging: 1 day

Final documentation: 1 day
